Autonomous Mode

Launches Claude in fully autonomous mode with all permissions bypassed, auto-compact enabled, extended thinking on, and agent teams active. Provide a task and Claude will work on it independently, restarting automatically when it exits.

The initial task Claude will receive when it starts. Pre-populated from the /go command — edit to customize.

When Claude exits, automatically restart with the same task after a 3-second delay. Press Ctrl+C during the delay to stop the loop.

Session

Resumes the most recent conversation exactly where it left off, restoring the full context. Claude sees everything from the previous session.

Starts a new conversation but pre-loads a summary of the most recent session. Useful when the previous context is too large or you want a fresh start with some history.

Shows detailed output including tool calls, file reads, and internal reasoning steps.

Runs a single prompt and prints the response to stdout, then exits. No interactive session. Great for scripting or quick queries.

Model & Effort

Which Claude model to use. Opus is the most capable but slowest. Sonnet is a good balance. Haiku is the fastest and cheapest. Opus Plan uses Opus for planning and Sonnet for execution.

Controls how much effort Claude spends on responses. Low gives quick, concise answers. High gives the most detailed and thorough responses.

If the primary model is unavailable or rate-limited, Claude CLI will automatically fall back to this model instead of failing.

Permissions & Tools

Controls what Claude can do without asking permission. Default prompts for each action. Accept Edits auto-approves file reads and writes but asks for shell commands. Plan is read-only. Don't Ask auto-approves everything except destructive operations. Bypass All disables all permission checks.

Restrict Claude to ONLY these tools (comma-separated). Example: Read,Grep,Glob would make Claude read-only. If set, Claude cannot use any tools not listed here.

Block Claude from using these specific tools (comma-separated). Example: Bash,Write prevents shell commands and file writes. Claude can still use all other tools not listed.

Context

Gives Claude a private scratchpad to think through complex problems before responding. Uses extra tokens but significantly improves reasoning on hard tasks like debugging, architecture decisions, and multi-step logic.

Maximum tokens for internal thinking (1,024 - 63,999). Higher means deeper reasoning but slower. 31,999 is a good default.

When context window usage reaches this percentage, Claude automatically compacts the conversation to free space. 80% is recommended. Lower (60-70%) for safety on long tasks. Higher (90%+) to maximize context.

Custom instructions appended to Claude's system prompt for this session. Use for project-specific rules, coding style preferences, or task context that Claude should always follow.

Project

Enables the multi-agent team system. Selected agent definitions are deployed to the project's .claude/agents/ directory on launch. If none are selected, defaults to the core team. Sentinel (context watchdog) is always active.
